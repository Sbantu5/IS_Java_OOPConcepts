Real-world objects contain __state_ and _behavior__.
A software object's state is stored in _fields__.
A software object's behavior is exposed through _methods__.
Hiding internal data from the outside world, and accessing it only through publicly exposed methods is known as data _encapulation__.
A blueprint for a software object is called a _class__.
Common behavior can be defined in a __superclass_ and inherited into a __subclass_ using the _extends__ keyword.
A collection of methods with no implementation is called an _Interface__.
A name space that organizes classes and interfaces by functionality is called a _package__.
The term API stands for __Application Programming Interface_?
